## Key Implementation Details

### Browser Support

- Chromium 147.0.7727.15 (Playwright v1217)
- Headless mode configured
- Supports both desktop and mobile viewports

### Locale Detection

- App uses i18next language detector (browser locale)
- Tests detect English locale (renders English UI)
- Language switching tested via i18n API
- Fallback language: Vietnamese

### Test Architecture

- **Strategy:** Diff-aware mapping with full suite execution
- **Test Runner:** Playwright Test (@playwright/test)
- **Web Server:** Vite preview on port 4173
- **Config:** `playwright.config.ts` with reuseExistingServer enabled

### Element Selectors

- Text matchers: `page.locator("text=...")`
- Heading matchers: `page.locator("h1:has-text(...)")`
- Button matchers: `page.locator("button:has-text(...)")`
- Attribute matchers: `page.locator("button:has(i.fa-cog)")`
- Bottom sheet title: `page.locator("h3:has-text(...)")`

### Waiting Strategies

- Network idle: `await page.waitForLoadState("networkidle")`
- Visibility: `await expect(locator).toBeVisible({ timeout: 5000 })`
- Keyboard: `await page.keyboard.press("Escape")`
- Delays: `await page.waitForTimeout(300)` for animations

## Issues Encountered & Resolutions

### Issue 1: Locale Detection

**Problem:** Tests initially expected Vietnamese text, but app detected English locale.
**Root Cause:** i18next-browser-languagedetector detected English environment.
**Resolution:** Updated all text selectors to use English translations instead.
**Impact:** Tests now explicitly validate language detection functionality.

### Issue 2: RegExp Locators

**Problem:** Playwright doesn't support regex patterns in `page.locator()`.
**Root Cause:** Used `/Turn \d+/` regex pattern (invalid).
**Resolution:** Changed to text matcher `page.locator("text=Turn")`.
**Impact:** Improved maintainability and reliability.

### Issue 3: Sheet Title Selector

**Problem:** Settings/skill sheets weren't found with `h2` selector.
**Root Cause:** BottomSheet component renders title in `<h3>` tag, not `<h2>`.
**Resolution:** Updated selectors to `h3:has-text('...')`.
**Impact:** All modal-based tests now pass.
